The `nono_py` module provides functions for applying sandboxes, starting the network proxy, and checking platform support.

It also provides policy-loading helpers for filesystem and network policy resolution.

## apply

```python
apply(caps: CapabilitySet) -> None
```

Apply the sandbox with the given capabilities. **This is irreversible.**

Once called, the current process and all child processes can only access resources granted by the capabilities. There is no way to expand permissions after this call.

<ParamField path="caps" type="CapabilitySet" required>
  The capability set defining permitted operations.
</ParamField>

**Raises:**
- `RuntimeError` - Platform not supported or sandbox initialization failed

### Example

```python
from nono_py import CapabilitySet, AccessMode, apply

caps = CapabilitySet()
caps.allow_path("/tmp", AccessMode.READ_WRITE)
caps.block_network()

# After this call, the process is sandboxed
apply(caps)

# These work:
with open("/tmp/test.txt", "w") as f:
    f.write("Hello")

# These fail with PermissionError:
open("/etc/passwd", "r")
```

<Warning>
**No escape hatch.** Once `apply()` succeeds:
- Permissions cannot be expanded
- The sandbox persists until process exit
- All child processes inherit the same restrictions
- There is no "undo" or "disable" function
</Warning>

### Error Handling

```python
from nono_py import CapabilitySet, AccessMode, apply, is_supported

caps = CapabilitySet()
caps.allow_path("/tmp", AccessMode.READ_WRITE)

if not is_supported():
    print("Warning: sandboxing not available, running without protection")
else:
    try:
        apply(caps)
        print("Sandbox applied successfully")
    except RuntimeError as e:
        print(f"Failed to apply sandbox: {e}")
        exit(1)
```

## sandboxed_exec

```python
sandboxed_exec(
    caps: CapabilitySet,
    command: list[str],
    cwd: str | None = None,
    timeout_secs: float | None = None,
    env: list[tuple[str, str]] | None = None,
    inherit_env: bool = False,
) -> ExecResult
```

Run a command in a sandboxed child process. The parent process stays unsandboxed, so it can run multiple child commands with different capability sets.

<ParamField path="caps" type="CapabilitySet" required>
  Capability set applied in the child before executing `command`.
</ParamField>

<ParamField path="command" type="list[str]" required>
  Command and arguments. The list must not be empty.
</ParamField>

<ParamField path="cwd" type="str | None">
  Working directory for the child.
</ParamField>

<ParamField path="timeout_secs" type="float | None">
  Maximum runtime in seconds. `None` means no timeout.
</ParamField>

<ParamField path="env" type="list[tuple[str, str]] | None">
  Environment variables for the child. By default, the parent environment is not inherited.
</ParamField>

<ParamField path="inherit_env" type="bool">
  If `True`, inherit the parent environment and apply `env` as overrides. Dynamic-loader variables such as `LD_*` and `DYLD_*` are rejected.
</ParamField>

**Returns:** `ExecResult` with `stdout`, `stderr`, and `exit_code` byte/output fields.

**Raises:**
- `ValueError` - Empty command or negative timeout
- `RuntimeError` - Fork or execution setup failed

### Example

```python
from nono_py import AccessMode, CapabilitySet, sandboxed_exec

caps = CapabilitySet()
caps.allow_path("/workspace", AccessMode.READ_WRITE)
caps.block_network()

result = sandboxed_exec(
    caps,
    ["python", "agent.py"],
    cwd="/workspace",
    env=[("PYTHONUNBUFFERED", "1")],
    timeout_secs=30.0,
)

if result.exit_code != 0:
    raise RuntimeError(result.stderr.decode("utf-8", errors="replace"))
```

<Tip>
Use `sandboxed_exec()` for supervisor-style programs that need to keep managing sessions, proxies, snapshots, or multiple child runs. Use `apply()` only when intentionally sandboxing the current process permanently.
</Tip>

## is_supported

```python
is_supported() -> bool
```

Check if sandboxing is supported on this platform.

**Returns:** `True` if sandboxing is available.

### Example

```python
from nono_py import is_supported

if is_supported():
    print("Sandbox available")
else:
    print("Sandbox not available")
```

### Platform Requirements

| Platform | Requirement |
|----------|-------------|
| Linux | Kernel 5.13+ with Landlock enabled |
| macOS | macOS 10.5+ (always available) |
| Windows | Not supported (always returns `False`) |

<Tip>
Use `support_info()` for more detailed information about why sandboxing might not be available.
</Tip>

## support_info

```python
support_info() -> SupportInfo
```

Get detailed information about sandbox support on this platform.

**Returns:** `SupportInfo` object with platform details.

### Example

```python
from nono_py import support_info

info = support_info()

print(f"Platform: {info.platform}")
print(f"Supported: {info.is_supported}")
print(f"Details: {info.details}")
```

Output examples:

**macOS:**
```
Platform: macos
Supported: True
Details: Seatbelt sandbox available
```

**Linux with Landlock:**
```
Platform: linux
Supported: True
Details: Landlock ABI v5 available
```

**Linux without Landlock:**
```
Platform: linux
Supported: False
Details: Landlock not available (kernel too old)
```

### Detailed Platform Check

```python
from nono_py import support_info
import sys

def require_sandbox():
    """Exit if sandboxing is not available."""
    info = support_info()

    if info.is_supported:
        return

    print(f"Error: Sandboxing required but not available", file=sys.stderr)
    print(f"Platform: {info.platform}", file=sys.stderr)
    print(f"Details: {info.details}", file=sys.stderr)

    if info.platform == "linux":
        print("\nTo enable Landlock on Linux:", file=sys.stderr)
        print("  1. Upgrade to kernel 5.13 or later", file=sys.stderr)
        print("  2. Ensure CONFIG_SECURITY_LANDLOCK=y in kernel config", file=sys.stderr)

    sys.exit(1)

require_sandbox()
```

## start_proxy

```python
start_proxy(config: ProxyConfig) -> ProxyHandle
```

Start the nono network filtering proxy.

Creates a localhost proxy server that provides domain filtering, credential injection, and audit logging for sandboxed child processes. The proxy runs on a background thread and is shut down via the returned handle.

<ParamField path="config" type="ProxyConfig" required>
  Proxy configuration including allowed hosts, credential routes, and bind settings.
</ParamField>

**Returns:** `ProxyHandle` with `env_vars()`, `credential_env_vars()`, `sandbox_env()`, `drain_audit_events()`, and `shutdown()`.

**Raises:**
- `RuntimeError` - Proxy failed to start (port in use, etc.)

### Example

```python
from nono_py import AccessMode, CapabilitySet, ProxyConfig, RouteConfig, start_proxy, sandboxed_exec

config = ProxyConfig(
    allowed_hosts=["api.openai.com"],
    routes=[
        RouteConfig(
            prefix="/openai",
            upstream="https://api.openai.com",
            credential_key="openai-key",
        ),
    ],
)
proxy = start_proxy(config)

# Inject env vars into sandboxed child
caps = CapabilitySet()
caps.allow_path("/workspace", AccessMode.READ_WRITE)
caps.proxy_only(proxy)
env = proxy.sandbox_env()
result = sandboxed_exec(caps, ["python", "agent.py"], cwd="/workspace", env=env)

# Collect audit trail and shut down
events = proxy.drain_audit_events()
proxy.shutdown()
```

## load_policy

```python
load_policy(json: str) -> Policy
```

Parse a policy JSON document into a `Policy` object.

**Returns:** `Policy` with group inspection and resolution helpers.

**Raises:**
- `ValueError` - Invalid policy JSON, including unknown policy fields

### Example

```python
from nono_py import load_policy

policy = load_policy(
    """
    {
      "groups": {
        "offline": {
          "description": "Disable outbound network",
          "network": { "block": true }
        },
        "proxy_web": {
          "description": "Allow only example.com through the proxy",
          "network": { "allow_proxy": ["example.com"] }
        }
      }
    }
    """
)

print(policy.group_names())
proxy_config = policy.resolve_proxy_config(["proxy_web"])
print(proxy_config.allowed_hosts if proxy_config else [])
```

`Policy.resolve_proxy_config()` follows the same network field naming used by
the main nono profile format:

- Canonical key: `network.allow_domain`
- Accepted aliases: `network.allow_proxy`, `network.proxy_allow`

If multiple groups specify different `upstream_proxy` / `external_proxy`
values, resolution fails with `ValueError` instead of silently overwriting the
earlier proxy.

Policy JSON is parsed strictly. Unknown fields are rejected instead of being
ignored.

When resolving groups with proxy-filtered network policy, `resolve_groups()`
fails closed by blocking direct network access. After starting the proxy, call
`caps.proxy_only(proxy)` and pass `proxy.sandbox_env()` to `sandboxed_exec()`.

## apply_unlink_overrides

```python
apply_unlink_overrides(caps: CapabilitySet) -> None
```

Apply post-resolution unlink overrides for writable paths. This helper is used after resolving policies that require unlink behavior to be tightened around writable grants.

### Example

```python
from nono_py import CapabilitySet, apply_unlink_overrides, load_policy

policy = load_policy(policy_json)
caps = CapabilitySet()
resolved = policy.resolve_groups(["workspace"], caps)

if resolved.needs_unlink_overrides:
    apply_unlink_overrides(caps)
```

## validate_deny_overlaps

```python
validate_deny_overlaps(deny_paths: list[str], caps: CapabilitySet) -> None
```

Validate final deny paths against a resolved capability set. Raises `ValueError` if a denied path is still covered by a filesystem grant.

### Example

```python
from nono_py import CapabilitySet, load_policy, validate_deny_overlaps

policy = load_policy(policy_json)
caps = CapabilitySet()
policy.resolve_groups(["default"], caps)
deny_paths = policy.resolve_deny_paths(["default"])
validate_deny_overlaps(deny_paths, caps)
```
